Thymeleaf tutorial - Exercise 9: more on conditions

This exercise introduces concepts from chapter 7 of the Using Thymeleaf manual

  1. Open the WEB-INF/templates/exercise09/answer.html template file with your code editor.
  2. A List of Customer objects has been set into the context model with variable name customerList. This is a complex bean, containing properties of the Gender and PaymentMethod enum types.
  3. Modify the template in order to practice three different kinds of condition expressions. Use the comment hints at the HTML to guide you.
    Tip: use name() in order to obtain the String representation of an enum value.
    • In the gender column you must show a boy image, a girl image or an 'Unknown' text depending on customer gender.
    • In the payment method column you must show a warning message when payment is not credit card payment.
    • In the balance column you must add a specific CSS class when balance is above $10000.
  4. Check your answer: Execute it.
  5. Now modify the template in order to obtain a more real-looking static prototype by adding some additional rows to the table (these rows should be removed when Thymeleaf executes).
  6. Check your answer again: See it statically | Execute it.
  7. Finally, you can compare your answer with our proposed solution: Execute it | See the code | See it statically.
